## Extra Credit 1: Prompt Trace Audit

### Folder Structure
Use this structure for your deliverables:

```text
extra/extra1/
├── code_deliverable/
├── log_deliverable/
└── vibe-report.md
```

---

## The Task

Compare two vibe coding tools or two prompt styles on the same small task.

1. Pick one simple feature to build or one page to improve.
2. Run the same task in two different tools, or with two different prompt styles in the same tool.
3. Record what each tool or prompt produced.
4. Note how the instruction stack, defaults, or workflow changed the outcome.

This assignment is about evidence, not hype. Focus on what you can actually observe.

## Submission: The Vibe Report

Document your work in `vibe-report.md`.

Your report should include:
1. The task you used for comparison.
2. The tools or prompts you compared.
3. What each one did well and poorly.
4. Which one gave you more control and why.
5. A link to your `log_deliverable/history.md` or other trace.
6. The course authenticity statement.

### Deliverables
- `code_deliverable/` with the artifact or mini feature you built
- `log_deliverable/` with transcripts, screenshots, or notes
- `vibe-report.md` with your analysis and reflection

---

*The goal is to learn how tools shape your work, not just which one looks best.*
