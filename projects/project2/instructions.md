# Project 2 — Build for a Human Value

**Type:** Group project (teams of 3–4) · **Weight:** 15% · **Assigned:** Thu, Jul 23 · **Due:** Mon, Jul 27 *before class*

Week 2 is about ethics: values embedded in tools (Mon), where safety and ethics
diverge (Tue), and using AI against AI harms (Wed). Now build something that
**purposefully supports a human value** — one your team chooses and can defend.

## The Task
Pick a **value** — sustainability, accessibility, privacy, fairness, dignity,
autonomy, honesty, care, … — and ship a small hosted app that *operationalizes* it.
Your team must be able to answer three questions clearly:

1. **What value** are you serving, and for **which user**?
2. **How** does the design advance it (not just claim it)?
3. **How would you know** it's actually working?

> [!TIP]
> Value ≠ topic. "A recycling app" is a topic. "Helps a renter with no car find the
> *nearest* drop-off so sustainability isn't gated on privilege" is a value made concrete.

This project is **user-tested in Week 3** — build something a classmate can actually try.

## Deliverables
Inside `projects/project2/`:
1. **Hosted app** — source in `code_deliverable/`, live link in your report.
2. **AI interaction log** — `log_deliverable/history.md`.
3. **Project report** — `project-report.md` from [the template](../project-report-template.md), with the **human-written ethical reflection** naming the value, the user, and the tensions.
4. **User-test writeup** — `user-test.md` from [the template](../user-test-template.md), completed after the Week 3 sessions.

Graded against the [Project Rubric](../rubric.md).
