# Activity Outline

| Theme | Weekday | Activity Title | Short Description | Readings | Detailed Description |
| :---- | :---- | :---- | :---- | :---- | :---- |
| Vibe Coding: Gaining Control | Week 1 Monday | **Personal Portfolio / About Me** | Set up Antigravity + GitHub and vibe code a hosted "about you" page — then debate whether copying a classmate's structure counts as plagiarism when an AI did the work. | [Example Portfolio](week1/activity1_example/index.html) | **Activity: Personal Portfolio / About Me.** Welcome to the course. Students will set up Antigravity and GitHub. <br><br> **Task:** Create an 'About Me' page. Must include: Who you are, hobbies, and a 'Professional' section. <br><br> **The Moral Twist:** We encourage 'stealing' or copying a fellow student's or the instructor's portfolio structure. Is it plagiarism if you vibe code it? Discuss attribution vs. inspiration. <br><br> **Deliverable:** Link to hosted GitHub Page and a Reflection in the vibe log. |
|  | Week 1 Tuesday | **Prompts that Steer the Vibe** | Run the same task through 2+ vibe coding tools with varied prompt strategies; document which prompting moves change the output the most and why. | [Why Johnny Can't Prompt](https://dl.acm.org/doi/10.1145/3544548.3581388) | Compare different vibe coding tools and prompt strategies on a shared task. Vary specificity, persona, examples, and constraints; collect outputs side-by-side; reflect on which prompt moves shifted quality vs. style vs. correctness. |
|  | Week 1 Wednesday | **Bias in Vibe Coding** | Probe several vibe coding tools for biased defaults — whose UI patterns, stock images, names, and example data show up when you don't specify? |  | Pick a generic prompt (e.g., "build a dating app", "build a hiring tool") and run it across multiple vibe coding tools. Catalog the defaults: image choices, name placeholders, example users, color palettes, recommended copy. Reflect on whose worldview is the unmarked default. |
|  | Week 1 Thursday | **Project 1 — Vibe Code Something You Want or Need** | Build a small hosted project for yourself this week, applying the strongest tool + prompt combo from days 1–3. |  | **Project 1.** Pick a real problem of your own and ship a small hosted project addressing it. Use the tool + prompting approach you found most effective in W1 Mon–Wed. Deliverable: hosted link + vibe report documenting tool choice, prompt log, and what broke. Due Monday Week 2 before class. |
|  |  |  |  |  |  |
| Ethics: Doing the right thing | Week 2 Monday | **Values in Vibe Coding (Light) — AI as Moral Assistant** | Lightweight intro: what moral, ethical, and political values get pushed for and embedded in technology? Then build a Vibe Coding Footprint Calculator that uses *real validated* CO₂/energy/water data for your last week of AI use and compares it against meal references (e.g., tofu vs. beef). |  | **Activity: AI as Moral Assistant + Footprint Calculator.** Short discussion: every tool embeds values (efficiency, attention, scale, frugality, sustainability, autonomy). Which ones are pushed by the vibe coding tools you already use? <br><br> **Task:** Use AI to help yourself act more ethically. Vibe code a personal calculator that ingests your week of AI usage (token / request counts) and translates it into CO₂, energy, and water using validated data sources (cite them). Compare against the footprint of your meals over the same week — e.g., tofu vs. beef. Optional precedent: instructor's protein-suffering calculator. <br><br> **Deliverable:** Hosted calculator + vibe report reflecting on what changed in your behavior (or didn't) once the numbers were visible. |
|  | Week 2 Tuesday | **Red-Teaming AI Safety: Dark Patterns On Purpose** | Adversarially push your tools to produce dark patterns and manipulative UI; document where the guardrails kick in, where they don't, and where "safety" and "ethics" diverge. |  | **Activity: Red-Team / Dark Patterns.** Take a benign-looking project brief and progressively push the AI to produce manipulative interfaces (forced continuity, confirmshaming, roach motels, etc.). Log refusals, partial refusals, and successes. Reflect: which harms does the model treat as "safety" vs. as design preference? Where does that distinction fail? |
|  | Week 2 Wednesday | **AI Against AI** | Use AI to defend against AI harms — poison scrapers with hidden facts, build privacy obfuscation, or protect against work-automation surveillance. | [Obfuscation Example](week2/7_22/examples/activity1_obfuscation-example/README.md) | **Activity: Poisoning AI Scrapers.** Implement obfuscation against data scraping by injecting fabricated but plausible facts into your site. <br><br> **Task:** Hide fake facts on your website, making them invisible to humans but visible to AI scrapers. Verify ingestion using `curl`. <br><br> **Deliverable:** Link to hosted site and vibe report reflecting on the technique. |
|  | Week 2 Thursday | **Project 2 — Build for a Human Value** | Make something that purposefully supports a chosen human value — yours, or one you found missing in this week's exercises. |  | **Project 2.** Pick a value (sustainability, accessibility, privacy, fairness, dignity, autonomy, …) and ship a small hosted project that operationalizes it. Articulate the value, the user, and how you'd know it's being served. Deliverable: hosted link + vibe report. Due Monday Week 3 before class. |
|  |  |  |  |  |  |
| Outcomes: Designing systems that are useful and empowering | Week 3 Monday | **User-Centered Evaluation (Half 1)** | Half the class runs live user tests of their projects with classmates; the other half observes and acts as users. Document what users actually do vs. what you assumed. |  | **Activity: Live User Evaluation, Round 1.** Group A demos their Project 2 to Group B and watches them use it without intervening. Group B captures observations, surprises, and breakdowns. Frame: user-centered vs. human/humanity-centered design — what do people *need* from this technology, vs. what does the technology assume they want? |
|  | Week 3 Tuesday | **Values at Play — Full Canvas + Evaluation Switch** | Groups switch user-test roles from Monday; in parallel, work the full VAP discovery canvas on your final project to surface the values you're embedding (and missing). |  | **Activity: VAP Discovery + Round 2 of User Evaluation.** Apply the full Values at Play canvas (from the [PIT Tech Ethics workshop](https://pitechethics.github.io/)) to your final project: identify embedded values, conflicts, and stakeholders. Round 2 of user testing with roles reversed. Synthesize: did user observations match the values you thought you were embedding? |
| Final Project | Week 3 Wednesday | **Final Project Working Day** | Open studio. Iterate on user-test findings + VAP analysis. Office hours throughout. |  | Independent work session. Instructors and TAs available all day for office hours and tool/setup help. No new content. |
|  | Week 3 Thursday | **Final Project Presentations** | Present a project that benefits someone outside the class — and that society keeps benefiting from after the course ends. |  | **Final Project.** Group project presentations. Each team shows the artifact, the values it operationalizes, evidence from user testing, and a credible plan for who keeps using/maintaining it after class. Graded against the project rubric in [syllabus.md](syllabus.md). |
