# Group Projects — Ethical Vibe Coding (TECHIE 1121)

This folder holds the **three group projects** for the course. They are kept
separate from the daily in-class activities (which live in `week1/`, `week2/`,
`week3/`) because projects are **team deliverables** graded against the
[Project Rubric](rubric.md), not the activity rubric.

| Project | Theme | Assigned | Due | Weight |
| :--- | :--- | :--- | :--- | :--- |
| [Project 1](project1/instructions.md) | Vibe code something your team wants or needs | Thu, Jul 16 (Wk1) | Mon, Jul 20 before class | 15% |
| [Project 2](project2/instructions.md) | Build for a human value | Thu, Jul 23 (Wk2) | Mon, Jul 27 before class | 15% |
| [Final Project](final/instructions.md) | Benefit someone outside the class — durably | Wed, Jul 29 (Wk3) | Thu, Jul 30 (presentations) | 25% + 15% presentation |

## How teams work

- **Team size:** 3–4 students. You keep the same team for all three projects unless instructors say otherwise.
- **One team repo:** Pick one member's fork (or a shared team repo) as the team's working repo. Everyone commits there so the AI-interaction log and git history reflect the whole team's work.
- **Everyone codes:** This is vibe coding — rotate who drives the AI. The prompt log should show more than one person's voice.
- **Hosting:** GitHub Pages for static apps; Vercel for framework apps (React/Vue/Next). Put the live link in your report.

## What every project deliverable contains

Each `projectN/` folder mirrors the activity structure so the automated site and
the Vibe-Trace logging keep working:

```text
projectN/
├── instructions.md          # the brief (read this first)
├── deadline.json            # due date
├── code_deliverable/        # your hosted app source
├── log_deliverable/         # history.md — AI interaction log (Vibe-Trace)
├── project-report.md        # your reflection (copy from ../project-report-template.md)
└── user-test.md             # user-testing writeup (copy from ../user-test-template.md)
```

The **Final Project** also has a `presentation/` folder for slides or a demo video.

## Grading

- Projects are graded against the [Project Rubric](rubric.md) (mirrored from the syllabus).
- **Deep ethical reflection is 25% and AI writing is impermissible there** (grammar-checking only).
- See each project's `instructions.md` for the exact required deliverables.
- Course-material AI provenance is documented in [`../COLOPHON.md`](../COLOPHON.md).
